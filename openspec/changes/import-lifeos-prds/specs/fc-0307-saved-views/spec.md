## Source PRD

# FC-0307 - Saved Views

## Summary

Saved Views let users reuse common filters and queue-like perspectives.

## User Need

Creators need quick access to recurring views such as ready for mockups, needs metadata, rejected ideas, or a specific campaign.

## Requirements

- Users can save a useful filtered view.
- Saved views can include filters such as status, tag, store, niche, group, or missing information.
- Users can rename or delete saved views.
- Saved views should update based on current matching data.
- Saved views should help repeat common workflows.

## Acceptance Criteria

- A user can save a filter combination.
- A user can reopen a saved view later.
- The saved view reflects current matching items.
- A user can remove views they no longer need.

## Out of Scope

- Shared team views
- Complex query language
- Dashboard widgets
- Scheduled reports
