## Why

Design Records preserve final artwork, but FusionCanvas cannot yet connect that artwork to the product preview images a creator uses to prepare a listing. FC-0206 adds Manual Mockup Records as the item-bound Listing-stage entity that tracks mockup images, the product/template/color/view they represent, the design they showcase, and whether they were manually attached or generated by the Basic Listing Tool, so creators can review available mockups before listing or publishing work.

## What Changes

- Add a Mockup domain entity owned by a listing, scoped to the Listing stage, with stable identity, timestamps, and optional reference to the design it showcases.
- Allow a mockup to reference one or more asset files through the existing asset-management capability, with Mockup added as a valid asset context kind.
- Track product type, vendor product, template, color variant, view, notes, and intended marketplace use as documented mockup metadata.
- Identify whether a mockup was manually attached or generated by the Basic Listing Tool.
- Allow multiple mockups per listing or design and preserve superseded mockups without deletion.
- Preserve enough metadata on generated mockups to be regenerated when the source design or template changes (full generation behavior arrives with FC-0212).
- Treat mockups as dependent records for listing and design deletion guarding.

## Capabilities

### New Capabilities

- `mockup-records`: Defines item-bound Listing-stage mockup records that connect mockup assets to listings and designs, track product/template/color/view metadata, distinguish manual from generated mockups, preserve superseded mockups, and store regeneration metadata for generated mockups.

### Modified Capabilities

- `core-domain-model`: Adds the Mockup entity to the core domain model and narrows the premature-advanced-entities exclusion so mockup records are now an accepted Phase 2 concern alongside concept and design versioning, while marketplace products, performance data, plugin data records, and custom workflow models remain out of scope.
- `asset-management`: Adds Mockup as a valid asset context kind alongside listing, niche, group, store, concept, and design.

## Impact

- Adds a Mockup domain entity, application contracts and orchestration for create/edit/attach/replace/supersede/remove, and SQLite persistence mappings through the existing snapshot boundary with a forward-only schema migration.
- Extends the asset context-link kind enum to include Mockup.
- Reuses `asset-management` for asset links, `design-records` for the showcased-design reference, `concept-versions` for inherited context, and `listing-management` for item identity and deletion guarding.
- Treats mockups as dependent creative records for listing and design deletion guards.
- Adds domain, application, integration, app, and UI tests for item-binding, multi-mockup preservation, manual/generated distinction, metadata, asset references, regeneration metadata, persistence, migration, and deletion-guard integration.
