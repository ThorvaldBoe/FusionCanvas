## Source PRD

# FC-0206 - Manual Mockup Records

## Summary

Manual Mockup Records track product preview images and generated mockup outputs for listing preparation.

## User Need

Creators need to connect mockups to the listings, designs, products, templates, and color variants they preview.

## Requirements

- Users can create mockup records.
- Mockups can be associated with listings and designs.
- Mockups can reference asset files.
- Mockups can include product type, vendor product, template, color, view, notes, and intended marketplace use.
- Mockups can identify whether they were manually attached or generated by the Basic Listing Tool.
- Multiple mockups can exist for one listing or design.
- Generated mockups should preserve enough metadata to be regenerated if the source design or template changes.

## Acceptance Criteria

- A user can attach a mockup image to a listing.
- A user can identify which design a mockup represents.
- A user can identify which product, template, and color variant a mockup represents when that data exists.
- A user can review available mockups before listing or publishing work.
- Mockup records preserve context without requiring automation.

## Out of Scope

- Advanced realistic mockup rendering
- Print provider catalog import
- Publishing sync
