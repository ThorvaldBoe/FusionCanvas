## Source PRD

# FC-0707 - Improvement Suggestions

## Summary

Improvement Suggestions identify candidates for revision, retirement, re-export, metadata improvement, or new variations.

## Requirements

- The product can surface listings that may need action based on status, metadata, assets, or performance.
- Suggestions should explain why an item was surfaced.
- Users can accept, dismiss, defer, or convert suggestions into work.
- Suggestions should not make changes without approval.
- Suggestion history can help avoid repeated noise.

## Acceptance Criteria

- A user can see listings that may need improvement.
- Each suggestion includes a clear reason.
- A user controls whether action is taken.

## Out of Scope

- Autonomous editing
- Guaranteed sales optimization
- External ad recommendations
