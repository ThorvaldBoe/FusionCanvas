# FC-0201 - Idea Inbox

## Summary

The Idea Inbox captures raw product ideas before they are ready to become listings.

## User Need

Creators need a low-friction place to store rough ideas without deciding immediately where they belong.

## Requirements

- Users can create raw ideas quickly.
- Ideas can include title, notes, audience, phrase fragments, or visual direction.
- Ideas can be assigned to a store, niche, or group when known.
- Creating an idea from a selected topic places it in that topic by default.
- Ideas can remain unprocessed until reviewed.
- Ideas can be converted into listings.
- Ideas can move into Concept, Design, or Listing workflow stages when the creator already has enough information.
- Ideas can be archived or rejected without deletion.

## Acceptance Criteria

- A user can capture an idea without creating a full listing.
- A user can review unprocessed ideas later.
- A user can convert a promising idea into a listing.
- A user can create an idea from the current topic without manually choosing the parent.
- A user can skip ahead from idea capture when the work starts as a concept, phrase, or design direction.
- Rejected ideas remain available for reference.

## Out of Scope

- AI idea generation
- Idea scoring
- Bulk idea import
- Marketplace validation
