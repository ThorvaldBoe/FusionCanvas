# FC-0405 - Concept Refinement

## Summary

Concept Refinement helps improve raw ideas into clearer product concepts.

In the default experience, concept refinement happens through the Basic Concept Tool by improving one selected design triangle element at a time.

## Requirements

- Users can refine an idea or listing concept with AI assistance.
- Refinement should use active store, niche, topic path, current item, and relevant metadata by default.
- Refinement must operate on an existing item and current or proposed concept version.
- Refinement can improve the idea, phrase, or graphic direction in context of the other triangle elements.
- Refinement can suggest audience fit, risks, phrase direction, visual direction, and reasoning.
- Users can accept, edit, reject, or save alternatives.
- Refinement should preserve the original concept.
- Outputs can become concept versions.
- A promising alternate output can be saved as a new item in the current topic without leaving the Concept tool.
- AI refinement should not overwrite saved concept values without user approval.

## Acceptance Criteria

- A user can ask AI to refine a raw concept.
- A user can refine a concept within the current topic and niche context without restating that context.
- A user can target refinement at idea, phrase, or graphic.
- A user can save a refined concept version.
- A user can save an off-direction but useful suggestion as a new item.
- The original idea remains available.

## Out of Scope

- Automatic approval
- Image generation
- Marketplace scoring
