# FC-0208 - Design Triangle View

## Summary

The Design Triangle View helps users evaluate alignment between idea, phrase, and graphic.

In the default workflow, this view is the central interaction model of the Basic Concept Tool.

## User Need

Creators need to see whether a listing's core idea, verbal hook, and visual direction support each other.

## Requirements

- The view presents idea, phrase, and graphic direction together.
- Users can edit or refine each triangle element.
- The idea element is required for every design.
- The phrase element can be marked as not used for graphic-only designs.
- The graphic element can be marked as not used for text-only designs.
- Each triangle element can be selected as the target for refinement.
- The view should make gaps or weak alignment easier to notice.
- Notes can capture concerns, alternatives, or improvement ideas.
- The view should support human judgment without requiring AI.
- When AI scoring is available, the middle of the triangle can show a quality indicator, weak element, or readiness state.
- The score should support judgment rather than enforce decisions.

## Acceptance Criteria

- A user can review idea, phrase, and graphic direction in one place.
- A user can update weak elements.
- A user can indicate that phrase or graphic is intentionally absent.
- A user can record design triangle notes.
- The view helps evaluate creative fit before production work continues.
- Optional scoring can identify weak triangle elements without replacing user judgment.

## Out of Scope

- Mandatory AI critique
- Mandatory automated scoring
- Image analysis
- Aesthetic validation
