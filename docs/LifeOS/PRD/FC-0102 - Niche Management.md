# FC-0102 - Niche Management

## Summary

Niche Management lets creators organize store work by audience, theme, market, or creative direction.

In FusionCanvas, a niche is more than a folder. It is a knowledge container for the audience, style, humor, constraints, risks, and recurring patterns that guide future listings.

## User Need

As a Print on Demand creator, I need to organize work by niche so I can keep related product ideas, creative guidance, and listings together.

## Goals

- Let users create and maintain niches inside a store.
- Make niches the default top-level topic areas inside a store.
- Preserve useful audience and creative context.
- Support browsing, filtering, and organization by niche.
- Keep niche structure flexible as the creator learns.

## Requirements

- The user can create a niche inside a store.
- The user can rename a niche.
- The user can edit niche notes and context.
- The user can archive and restore a niche.
- A niche can contain groups.
- A niche can contain listings.
- A niche can describe audience, style, humor, constraints, legal risks, creative guidance, or other useful context.
- A niche should remain useful even before any listings exist inside it.

## User Workflows

### Create a Niche

The user creates a niche when they identify a target audience, theme, design family, or market segment worth organizing separately.

Examples include tabletop RPG players, fantasy taverns, cat owners, Halloween, grunge textures, or teacher gifts.

### Add Niche Context

The user adds notes that help guide future listing work.

This context may include audience traits, humor style, design constraints, successful patterns, ideas to avoid, or research notes.

### Organize Work by Niche

The user browses a store by niche, creates groups inside a niche, and places listings where they belong.

The niche should make the store easier to understand at a glance.

## Acceptance Criteria

- A user can create multiple niches in one store.
- A user can capture the difference between separate audiences or product themes.
- A user can store useful niche notes that will still matter later.
- A user can browse a store by niche and understand where work belongs.
- Archived niches are retained without cluttering active work.

## Out of Scope

- AI niche research
- Trend tracking
- Niche scoring
- Marketplace demand analysis
- Cross-store niche libraries
- Automated niche recommendations

## Open Questions

- How much structured niche context is useful in Phase 1 before the editor becomes too heavy?
- Should a listing be allowed to move between niches freely?
- Should archived niches hide their groups and listings from normal search by default?
